# Claude Scouter — 제품 계획서

## 한 줄 정의
> Claude Code 사용자를 위한 데스크톱 위젯. 내 AI 오케스트레이션이 의도대로 동작하는지 실시간으로 증명한다.

## 타겟 유저
- Claude Code (CLI) 파워 유저
- 멀티 세션/에이전트 오케스트레이션하는 개발자
- Max 플랜 쓰면서 한도 관리 필요한 사람

## 핵심 니즈

| 니즈 | 질문 | 현재 해결책 |
|------|------|------------|
| 의도 검증 | "시킨 대로 하고 있나?" | 없음 (터미널 일일이 확인) |
| 효율 증명 | "잘 쓰고 있나?" | 없음 |
| 이상 감지 | "뭔가 잘못된 건 아닌가?" | 토큰 다 써야 알게 됨 |

## 3가지 뷰 모드

### 🖥️ Full Dashboard (~800x600+)
모니터링 전용. 세컨드 모니터에 띄워놓기.
- 전체 토큰 게이지 + 상세 breakdown (input/output/cache)
- 세션 목록 + 각 세션 토큰 소비 타임라인 그래프
- 캐시 효율 (hit ratio), I/O 비율 차트
- 오케스트레이션 플로우 뷰 (세션 간 관계/순서)
- 히스토리/트렌드 (일별/주별)
- 효율 스코어 ("전투력")
- 알림 로그

### 📊 Compact Dashboard (~320x400)
옆에 띄워놓고 가끔 확인.
- 토큰 게이지 + 퍼센트
- 활성 세션 수 + 상태 dot (idle/active/error)
- 번레이트 + 남은 시간
- 이상 감지 알림 배지
- 클릭 → Full로 전환

### 🔴 Mini Widget (~80x80)
코딩하면서 구석에 띄워놓는 always-on 모드.
- 원형 게이지 하나 (사용량 %)
- 상태 색상 (초록/노랑/빨강)
- 번레이트 숫자
- 이상 시 깜빡임
- 클릭 → Compact로 확장
- always-on-top, 드래그 배치

### 뷰 전환
타이틀바 버튼 또는 더블클릭으로 Full ↔ Compact ↔ Mini 순환.

## 기술 스택
- **Frontend:** React + TypeScript + Vite
- **Desktop:** Tauri v2 (Rust)
- **데이터 소스:** Claude Code JSONL 로그 파일 watch (`~/.claude/`)
- **상태 관리:** zustand 또는 useState

## 데이터 수집 방식
Claude Code는 `~/.claude/` 아래에 세션 로그를 JSONL로 남김.
→ Rust 백엔드에서 파일 watch (notify crate)
→ 파싱 + 집계
→ Tauri event로 프론트엔드에 emit

## 차별점
1. **데스크톱 위젯** — 항상 떠있으면서 한눈에 (CLI 도구들과 차별)
2. **오케스트레이션 검증** — 단순 미터기가 아닌 "잘 쓰고 있는지" 증명
3. **이상 감지** — 문제를 토큰 다 쓰기 전에 잡아줌
4. **3단계 뷰** — 상황에 맞는 정보 밀도 선택

## 이름
Claude Scouter — 드래곤볼 스카우터처럼 한쪽 눈에 걸쳐놓고 전투력(효율)을 측정하는 도구.
